       8 OpenRevu is an instrument of thought for people who build things. I
         t captures intent, not just ink. Every markup carries meaning — typ
         e, status, and context travel with the mark from creation to resolu
         tion. Every view reveals progress — ambient counts, semantic colors
         , and filterable lists make the state of a document visible at a gl
         ance. The tool disappears so the work remains. A construction profe
         ssional reviewing plans should feel the clarity of knowing exactly
         what needs attention, the confidence that nothing was missed, and t
         he satisfaction of visible progress. We don't build drawing tools.
         We build a system that extends human cognition over spatial documen
         ts.

      12 ## Design Principles (Derived)
      13
      14 1. **Intent first, shape second.** The user declares meaning, then
         draws. Color and metadata follow automatically.
      15 2. **Color is information.** Red = issue, amber = question, green =
          approval, blue = change, gray = note. No arbitrary color assignmen
         t.
      16 3. **Markups are data, not drawings.** Every mark is a structured o
         bject: type, status, note, location, date, author. Queryable, sorta
         ble, exportable.
      17 4. **Ambient awareness over tab switching.** Progress counts, type
         summaries, and status indicators are always visible in peripheral v
         ision.
      18 5. **Narrow the gulfs.** Minimize steps between thought and action
         (execution gulf). Maximize visible feedback about system state (eva
         luation gulf).
      19 6. **Support deep work.** Never interrupt the user's cognitive thre
         ad. Auto-save, keyboard shortcuts, semantic defaults — the tool han
         dles the shallow work.

      23 ## Theoretical Foundations
      24

         on, Gulf of Evaluation, conceptual models, feedback, discoverabilit

         support for sustained attention, tool transparency
      27 - **Mihaly Csikszentmihalyi, *Flow*** — clear goals, immediate feed
         back, balance of challenge and skill, merging of action and awarene

         s, no agency without feedback, narrow interfaces
