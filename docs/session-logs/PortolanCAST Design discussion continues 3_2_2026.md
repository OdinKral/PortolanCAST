Events anchor time.

A structured support or refutation attached to a hypothesis.

- Experiment
    
- External document
    
- Replication
    

Evidence is not narrative.  
It is traceable support.

### 7. Model

An integrated, structured explanation across components and hypotheses.

Models are living artifacts.  
They version.

### 8. Agent

A decision-making entity.

Examples:

- Technician
    
- Researcher
    
- AI system
    

Agents create hypotheses and observations.

# 5. Typography

### Primary Typeface

A neutral geometric sans-serif:

- Inter
    
- IBM Plex Sans
    
- SF Pro (if Apple ecosystem)
    

Characteristics:

- Wide aperture
    
- Stable forms
    
- High legibility at small sizes
    
- No theatrical personality
    

---

### Secondary Typeface (Optional)

For long-form philosophical documents:

- A restrained serif (e.g., Source Serif)
    

Used sparingly.

---

# 6. Layout Philosophy

### Grid System

- Strict 8pt spacing system
    
- Strong vertical rhythm
    
- Left-aligned hierarchy
    
- Generous negative space
    

No asymmetrical chaos.

---

### Interface Structure

Every screen should feel like:

- A cockpit
    
- A research console
    
- A DAW timeline
    
- A systems dashboard
    

But never a “productivity app.”

---

# 7. Motion Principles

Motion should:

- Indicate causality
    
- Show flow direction
    
- Reveal structural shifts
    
- Teach systems thinking
    

Avoid:

- Bounce
    
- Playful easing
    
- Unnecessary animation
    

Motion = signal, not spectacle.

---

# 8. Information Hierarchy

Every element must declare:

1. Is this a component?
    
2. Is this a relationship?
    
3. Is this evidence?
    
4. Is this interpretation?
    
5. Is this hypothesis?
    

Visual cues:

- Solid line = confirmed
    
- Dashed line = hypothesis
    
- Dotted line = weak correlation
    
- Thick edge = strong influence
    

This is epistemic literacy embedded in UI.

---

# 9. Emotional Tone

The system should feel:

- Like entering a quiet control room at dawn
    
- Like calibrating instruments before flight
    
- Like preparing sails before wind
    

It should not:

- Overwhelm
    
- Gamify
    
- Infantilize
    
- Sell
    

---

# 10. Brand Voice

When writing:

- Declarative sentences
    
- Short paragraphs
    
- No exclamation marks
    
- No hype language
    
- No startup metaphors
    

Example:

> This relationship lacks supporting evidence.  
> Add documentation or mark as hypothesis.

# 7. Naming Convention

Use:

category.subcategory.role.state

Examples:

- `semantic.text.primary`
    
- `graph.edge.hypothesis`
    
- `component.panel.bg`
    

Avoid:

- Random color naming
    
- Hard-coded hex in components
    
- Style duplication
    

Entropy reduction applies to tokens.
